# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.x     | :white_check_mark: |

## Reporting a Vulnerability

We take the security of this SDK seriously. If you discover a security vulnerability, please follow these steps:

1. **Do not** disclose the vulnerability publicly until it has been addressed by our team.
2. Submit a detailed report to support@hashgraphonline.com.
3. Include:
   - A clear description of the issue
   - Reproduction steps
   - Potential impact
   - Suggested remediation (if available)

We acknowledge reports within 48 hours and aim to provide an initial assessment within 72 hours.

## Security Best Practices

When using this SDK:

1. Use the latest stable SDK version.
2. Keep private keys and credentials secure.
3. Use environment-based secrets management.
4. Audit dependencies regularly.
5. Apply least-privilege principles for accounts and keys.

## Security Updates

Security-related updates are delivered through normal releases and dependency updates.

## Contact

For security-related inquiries, contact support@hashgraphonline.com.
